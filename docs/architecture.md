# Agent-Retina-World 系统架构设计

## 1. 设计动机

桌面是开发者最高频的工作界面，但现有 Agent 缺乏对屏幕上下文的持续感知能力。Agent-Retina-World 的目标是：

- 低侵入地采集屏幕状态
- 用 VLM 将像素转化为结构化语义
- 将离散帧聚合为可查询的活动时间线
- 基于记忆提供总结、待办等主动服务

## 2. 分层架构

```
┌─────────────────────────────────────────────────────────┐
│                    主动服务层                            │
│         每日总结 · 待办推断 · 时间分布统计                │
├─────────────────────────────────────────────────────────┤
│                    记忆层                                │
│         事件记忆 (ActivityEvent) · 任务记忆 (Task)       │
├─────────────────────────────────────────────────────────┤
│                    认知层                                │
│    页面分类 · 文本块 · 实体 · 行为 · 高价值事件          │
├─────────────────────────────────────────────────────────┤
│                    感知层                                │
│              定时截图 · 感知哈希去重                      │
└─────────────────────────────────────────────────────────┘
```

## 3. 去重策略

| 阶段 | 方法 | 作用 |
| --- | --- | --- |
| L1 | 感知哈希 (pHash) | 毫秒级过滤静态/近重复帧 |
| L2 | 直方图余弦相似度 | 布局微变但内容近似的帧 |
| L3 | 语义 embedding | 基于窗口指纹/摘要向量的余弦相似度去重 |

预期效果：在静态桌面场景下，VLM 调用量可下降 50% 以上。

## 4. VLM 输出 Schema

```json
{
  "page_category": "ide",
  "text_blocks": ["..."],
  "entities": ["文件名", "函数名"],
  "user_action": "debugging",
  "high_value_events": ["保存文件", "运行测试"],
  "summary": "正在 IDE 中调试 Python 模块"
}
```

## 5. Activity 聚合规则

- 同 `page_category` + 同 `user_action`
- 相邻帧时间间隔 ≤ 合并窗口（默认 3 分钟）
- 输出 `ActivityEvent`：起止时间、摘要、证据截图路径列表

## 6. 技术栈

- Python 3.10+
- mss / Pillow / imagehash
- SQLite
- VLM / Chat：OpenAI 兼容 API（Qwen2.5-VL、gpt-5.4-mini 等）
